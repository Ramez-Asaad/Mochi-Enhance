# Training-Free Enhancements for Open-Weight Video Diffusion Models

This repository contains the complete experimental pipeline and results for our research on zero-cost, training-free enhancements for the Mochi-1-preview open-weight video diffusion model.

We present a methodology to significantly improve the **temporal consistency** (measured via SSIM/LPIPS) and **semantic alignment** (measured via CLIP-SIM) of generated videos without requiring any fine-tuning or external post-processing networks.

## 🚀 Key Results

By independently manipulating the Flow Match Euler Discrete Scheduler, injecting structured negative embeddings, and amplifying classifier-free guidance, we achieve:
- **1.3% absolute improvement in SSIM** (smoother, more stable temporal consistency)
- **21% reduction in LPIPS** (drastically fewer visual artifacts and morphing)
- **Zero computational overhead** (enhancements run natively during diffusion)
- Preserved or improved CLIP-SIM scores (strong prompt adherence)

For full numerical results and analysis, please refer to the `paper.tex` manuscript or the generated visualizations in `results/figures/`.

## 📁 Repository Structure

### 1. Core Pipeline
- `config.py`: Central configuration file. Defines the generation hyperparameters, the 5 diverse evaluation prompts, the 3 random seeds, and the 5 ablation configurations.
- `main.py`: The primary orchestration script. Automatically selects the optimal GPUs, applies the native model edits (Scheduler Shift, Negative Prompts, Guidance Scale), and runs the entire ablation study (75 experiments) in parallel using PyTorch multiprocessing.
- `eval.py`: The evaluation suite. Contains a lazily-loaded `EvalSuite` class that computes CLIP-SIM, SSIM, LPIPS, and Fréchet Video Distance (FVD) using pre-trained neural networks.
- `utils.py`: Contains general utilities, including legacy post-processing experiments (RAFT interpolation, Real-ESRGAN upscaling) that were discarded in favor of our superior native enhancement approach.

### 2. Analysis & Visualization
- `compute_fvd.py`: Standalone script to compute the Fréchet Video Distance (FVD) between the baseline and enhanced videos utilizing InceptionV3 feature extraction.
- `visualize.py`: Automatically parses the CSV results and generates publication-ready LaTeX tables, grouped bar charts, variance box plots, and per-prompt improvement heatmaps.
- `extract_frames.py`: Extracts representative frames from the generated videos (t=0, t=mid, t=end) and compiles them into a qualitative comparison grid.

### 3. Data & Artifacts
- `results/`: Contains all generated data.
  - `videos/`: Directory containing all 75 MP4 videos generated during the ablation study.
  - `raw_results.csv`: The untampered metric readings for every individual experiment.
  - `ablation_summary.csv`: The aggregated mean ± std metrics for each configuration.
  - `figures/`: The output directory for all PNG charts and the LaTeX table generated by `visualize.py`.
- `paper.tex`: The complete, submission-ready IEEE-format LaTeX manuscript detailing our methodology, experiments, and findings.

## 🛠️ Installation & Usage

### Requirements
- A multi-GPU environment is highly recommended (we tested on 2x RTX A6000 48GB).
- Python 3.10+
- PyTorch 2.0+

```bash
# Clone the repository
git clone https://github.com/Ramez-Asaad/selected-topics-in-AI-project.git
cd selected-topics-in-AI-project

# Install dependencies
pip install -r requirements.txt
pip install seaborn matplotlib pandas scipy imageio
```

### Running the Full Ablation Study
The pipeline is fully automated and designed for multi-GPU parallel execution. It will automatically query `nvidia-smi`, select the two GPUs with the most free VRAM, and split the 75 generation tasks between them.

```bash
python3 main.py
```

### Generating Visualizations
Once the ablation study completes, you can compute the FVD scores and generate the charts for the paper:

```bash
# 1. Compute FVD
python3 compute_fvd.py

# 2. Generate charts and LaTeX tables
python3 visualize.py

# 3. Generate qualitative frame grids
python3 extract_frames.py
```
